# Release Checklist — التجربة الأولى (0.1.0+trial-1)

قائمة تحقق قبل بدء **التجربة الأولى** مع المستخدمين.

راجع أيضاً: [`database/TRIAL_SETUP.md`](database/TRIAL_SETUP.md) · [`CHANGELOG.md`](CHANGELOG.md)

## 0) الإصدار والإشعار

- [ ] `web/src/config/app-release.ts` — `version` و`build` محدّثان
- [ ] `web/package.json` — `version` يطابق `app-release.ts`
- [ ] `CHANGELOG.md` — سجل التغييرات لهذا البناء
- [ ] إشعار «تجربة أولى» يظهر عند أول دخول (زر «فهمت» يخفيه حتى البناء التالي)

## 1) قاعدة البيانات (Supabase)

- [ ] مشروع Supabase **تجريبي** (ليس إنتاجاً)
- [ ] تشغيل `database/setup_all.sql` في SQL Editor
- [ ] التحقق باستعلامات `TRIAL_SETUP.md` §3 (أعمدة العملة، 7 حسابات جذر)
- [ ] *(اختياري)* `database/03_test_cases.sql` — توازن القيود
- [ ] إضافة الحسابات الفرعية التشغيلية المطلوبة
- [ ] `/vouchers/settings` — حسابات افتراضية لكل نوع سند

## 2) متغيرات البيئة

### محلي (`web/.env.local`)

- [ ] `NEXT_PUBLIC_SUPABASE_URL`
- [ ] `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`

### Vercel / CI

- [ ] Root Directory = `web`
- [ ] متغيرات Supabase مضبوطة
- [ ] `web-ci.yml` ينجح (lint + build)

## 3) المصادقة والمستخدمون

- [ ] Email/Password مفعّل في Supabase Auth
- [ ] تسجيل أول مستخدم → يصبح admin
- [ ] إكمال معالج `/setup` (أو تشغيل `patch_setup_complete.sql` على قاعدة قديمة فيها مستخدمون مسبقاً)
- [ ] إنشاء مستخدمي تجربة وصلاحياتهم من `/settings/permissions`

## 4) اختبار الوظائف (يدوي)

### دليل الحسابات

- [ ] عرض الشجرة وإضافة حساب فرعي
- [ ] فتح سند جديد — الحساب الجديد يظهر دون إعادة تحميل الصفحة

### السندات

- [ ] سند قبض — عملة، سعر صرف، اعتماد، ترحيل
- [ ] سند دفع — نفس المسار
- [ ] سند تصفية — مراكز كلفة متوازنة
- [ ] قائمة السندات — قيمة، اعتماد، ترحيل، تعديل
- [ ] بحث حساب في أسطر السند — القائمة أمام الجدول

### التقارير

- [ ] كشف حساب — عملة عرض، أعمدة أصلية منفصلة
- [ ] ميزان مراجعة — متوازن بعد السندات
- [ ] ربط السند ↔ القيد

### الإعدادات

- [ ] بيانات الشركة والشعار
- [ ] صفحة «عن البرنامج» تعرض الإصدار وملاحظات التجربة

## 5) جودة الكود

- [ ] `npm run lint` بدون أخطاء
- [ ] `npm run build` ينجح

## 6) ما قبل إبلاغ المختبرين

- [ ] مشاركة رابط التطبيق وبيانات الدخول
- [ ] توضيح أن البيانات تجريبية و`setup_all` يمسح القاعدة
- [ ] قناة لجمع الملاحظات (بريد / واتساب / نموذج)

## 7) بعد التجربة

- [ ] مراجعة السندات المرحّلة
- [ ] توثيق الأخطاء في Issues أو ملاحظات داخلية
- [ ] زيادة `build` في `app-release.ts` عند النشر التالي (مثل `trial-2`)

## حدود معروفة (لا تُعتبر أخطاء في trial-1)

- ميزان المراجعة لا يستخدم `debit_base`/`credit_base`
- القيود اليدوية بدون حقول عملة مخزنة
- لا يمكن تعديل مرفقات سند مرحّل
