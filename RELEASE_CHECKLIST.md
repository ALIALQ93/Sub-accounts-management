# Release Checklist — Sub Accounts Management

قائمة تحقق قبل التثبيت النهائي للمستخدمين.

## 1) قاعدة البيانات (Supabase)

- [ ] تشغيل `database/setup_all.sql` في SQL Editor (يحذف البيانات السابقة ويعيد الإعداد)
- [ ] *(اختياري)* تشغيل `database/03_test_cases.sql` والتحقق من توازن القيود (`total_debit = total_credit`)
- [ ] التأكد من وجود 7 حسابات رئيسية (الموجودات، الالتزامات، حقوق الملكية، المبيعات، المشتريات، المصاريف، الايرادات)
- [ ] إضافة الحسابات الفرعية المطلوبة للتشغيل الفعلي

## 2) متغيرات البيئة

### محلي (`web/.env.local`)

- [ ] `NEXT_PUBLIC_SUPABASE_URL`
- [ ] `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` (أو `NEXT_PUBLIC_SUPABASE_ANON_KEY`)

### GitHub Actions

- [ ] Variable: `SUPABASE_URL`
- [ ] Secret: `SUPABASE_PUBLISHABLE_KEY`

### Vercel

- [ ] Root Directory = `web`
- [ ] Framework = Next.js
- [ ] `NEXT_PUBLIC_SUPABASE_URL`
- [ ] `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`

## 3) CI/CD

- [ ] workflow `web-ci.yml` ينجح (lint + build)
- [ ] لا توجد تحذيرات حرجة في GitHub Actions
- [ ] Vercel deploy ينجح بدون 404

## 4) اختبار الوظائف (يدوي)

### دليل الحسابات

- [ ] عرض الشجرة
- [ ] إضافة حساب فرعي
- [ ] تعديل الاسم / postable
- [ ] تعطيل حساب

### العملاء والموردين

- [ ] إضافة / تعديل / تعطيل

### السندات

- [ ] إنشاء سند قبض (account mode)
- [ ] إنشاء سند دفع (account mode)
- [ ] إنشاء سند تصفية (invoice mode + allocations)
- [ ] حفظ → اعتماد → ترحيل
- [ ] عكس سند مرحّل

### القيود والتقارير

- [ ] القيد يُولَّد تلقائيًا بعد ترحيل السند
- [ ] رابط من السند إلى القيد والعكس
- [ ] فلترة تاريخ في القيود وميزان المراجعة
- [ ] ميزان المراجعة متوازن

### لوحة التحكم

- [ ] بطاقات الإحصائيات تظهر بيانات صحيحة
- [ ] آخر حركة تعرض السند/القيد الأحدث

## 5) جودة الكود

- [ ] `npm run lint` بدون أخطاء
- [ ] `npm run build` ينجح

## 6) ما قبل الإطلاق

- [ ] نسخة احتياطية من قاعدة البيانات
- [ ] توثيق بيانات الدخول (إذا أُضيفت لاحقًا)
- [ ] مشاركة رابط Vercel مع المستخدمين
- [ ] تحديد مسؤول للدعم الأولي بعد الإطلاق

## 7) بعد الإطلاق (مراقبة أسبوعية)

- [ ] مراجعة السندات المرحّلة يوميًا
- [ ] التحقق من توازن ميزان المراجعة
- [ ] متابعة أي أخطاء في Vercel Logs
- [ ] جمع ملاحظات المستخدمين للمرحلة التالية (Auth, RLS, PDF reports)
