# إعدادات المخزون العامة — الجرد والقيود والتكلفة

> **الموضوع:** خيارات **أول إعداد** للبرنامج — تحدد طريقة الجرد، توليد القيود، وتوزيع التكلفة.  
> **قرار حاسم:** **غير قابلة للتعديل** بعد وجود عمليات مخزنية/فواتير مرحّلة.  
> **آخر تحديث:** 2026-07-05 · **الحالة:** 🟡 موثّق من شرح المستخدم

---

## 1) مبدأ القفل (Lock)

| البند | القرار |
|--------|--------|
| متى تُختار | **معالج الإعداد الأولي** — قبل أي فاتورة/حركة مخزنية مرحّلة |
| بعد العمليات | **لا تعديل** على: طريقة الجرد، فصل التكلفة per مستودع/CC، نظام التكلفة الأساسي |
| السبب | تغييرها لاحقاً يكسر تسلسل التكلفة والقيود التاريخية |
| التحقق | `company_settings.inventory_locked = true` بعد أول `posted` يمس المخزون |

> مكان التخزين: [الإعدادات العامة عند التثبيت](../ملف/الإعدادات-العامة-عند-التثبيت.md) · `company_foundation_settings`

## 2) طريقة الجرد: دوري vs مستمر

### 2.1) التأثير على توليد القيود اليومية

**توليد القيود ينعكس وفق الجرد** — دوري أم مستمر. هذا محور القيود المحاسبية، وليس مجرد «عرض مخزون».

| الحدث | **جرد مستمر** (Perpetual) | **جرد دوري** (Periodic) |
|--------|---------------------------|---------------------------|
| **مبيعات** | قيد بيع + **قيد تكلفة**: مدين تكلفة مبيعات — دائن مخزون | قيد بيع (إيراد/ذمم) — **بدون** قيد تكلفة فوري |
| **مشتريات** | مدين مخزون — دائن مورد/نقد | مدين **مشتريات** (مصروف) — دائن مورد — أو حسب السياسة |
| **مناقلة** | مخزون هدف ↔ مصدر (+ **تكلفة إضافية** اختيارية §4) | كمية + مصاريف نقل §4.2 |
| **نهاية الفترة** (دوري) | — | قيد تسوية: تكلفة مبيعات ↔ مخزون فعلي |

### 2.2) نقطة مهمة: احتساب المخزون مع كل حركة

| البند | القرار |
|--------|--------|
| حتى في **الجرد الدوري** | النظام **يحتسب ويتتبع المخزون** (كميات وتكلفة نظامية) مع **كل حركة** |
| الفرق الجوهري | **ليس** إخفاء المخزون — بل **متى وكيف يُولَّد قيد اليومية** (فوراً vs آخر الفترة) |

```
                    ┌──────────────────────┐
                    │   حركة مخزنية        │
                    │   (كمية + تكلفة      │
                    │    نظامية دائماً)    │
                    └──────────┬───────────┘
                               │
              ┌────────────────┴────────────────┐
              ▼                                 ▼
     ┌─────────────────┐               ┌─────────────────┐
     │  جرد مستمر      │               │  جرد دوري       │
     │  قيد فوري       │               │  قيد مبيعات/    │
     │  (مبيعات+تكلفة) │               │  مناقلة مختلف   │
     └─────────────────┘               │  تكلفة آخر فترة │
                                       └─────────────────┘
```

## 4) تكلفة إضافية على المناقلة — دوري **و** مستمر

> **من شرح المستخدم:** في **نظامي** الجرد (الدوري والمستمر) يمكن **إضافة تكلفة** على المناقلة — نقل، أجور عمال، شحن داخلي…

| البند | القرار |
|--------|--------|
| الإتاحة | **كلا** `periodic` و `perpetual` |
| الفرق بينهما | **شكل القيد** — وليس إتاحة التكلفة الإضافية |

| نوع المصروف | أمثلة | التأثير على التكلفة |
|-------------|--------|---------------------|
| مصاريف نقل | شحن بين مستودعات | يُحمَّل حسب [§5](#5-فصل-التكلفة--مستودعات-ومراكز-الكلف) |
| أجور عمال | تحميل، تفريغ، مناولة | قد **يعيد تكليف** المادة أو يضيف على مستودع الهدف |

### 4.1) جرد **مستمر** + تكلفة إضافية

```
أساس:     مدين مخزون هدف — دائن مخزون مصدر  (بتكلفة المنقول)
إضافة:    مدين مخزون هدف — دائن مصروف نقل/أجور  (أو تضمين في تكلفة الوحدة المنقولة)
```

- التكلفة الإضافية **ترفع** تكلفة البضاعة في المستودع الهدف (أو عالمياً إن لا فصل per مستودع).

### 4.2) جرد **دوري** + تكلفة إضافية

```
أساس:     حركة كمية (تتبع نظامي دائماً §2.2)
إضافة:    أسطر/قيود مصاريف نقل — مدين مخزون أو مصروف — دائن نقد/ذمم
```

- قد **لا** يُولَّد قيد مخزون↔مخزون تلقائي — لكن **التكلفة الإضافية** تُدخل بنفس واجهة المناقلة (أسطر مصروف أو قيد مرافق).

### 4.3) واجهة المناقلة (مقترح)

| الطريقة | الوصف |
|---------|--------|
| **تبويب الحسابات الإضافية** على فاتورة المناقلة | **الافتراضي** — مصروف نقل/أجور كسطر حساب §13.3 |
| **قيد مرافق** | اختياري للمبالغ الكبيرة أو المعالجة المنفصلة |

> انظر [مستند الفاتورة §13](../الفواتير/تعريف-الفواتير.md).

## 5) الفرق الجوهري: مبيعات + شكل قيد المناقلة

| العملية | ما يتغيّر بين الدوري والمستمر |
|---------|--------------------------------|
| **قيد المبيعات** | المستمر: COGS فوري. الدوري: إيراد/ذمم فقط عند البيع |
| **قيد المناقلة (أساس)** | المستمر: مخزون↔مخزون. الدوري: غالباً كمية + مصاريف منفصلة |
| **تكلفة إضافية على المناقلة** | **متاحة في الاثنين** — §4 |

---

## 6) فصل التكلفة — مستودعات ومراكز الكلف

خيارات **عامة** (قفل §1) — يحددها المستخدم عند أول إعداد:

| الإعداد | `cost_per_warehouse` | `cost_per_cost_center` |
|---------|----------------------|-------------------------|
| **فصل per مستودع** | ✅ نعم | — |
| **فصل per مركز كلف** | — | ✅ نعم |
| **بدون فصل** | ❌ تكلفة موحّدة للمادة | ❌ |

### 6.1) فصل التكلفة لكل مستودع

| السلوك | الوصف |
|--------|--------|
| تكلفة المادة | قد تكون **مختلفة** في مستودع أ عن مستودع ب |
| المناقلة | نقل **بنفس تكلفة المصدر** — التكلفة تنتقل مع الكمية |
| مصاريف النقل | تُضاف على تكلفة **المستودع الهدف** (أو حسب سياسة التحميل) |

### 6.2) بدون فصل التكلفة + مناقلة بأجور

| السلوك | الوصف |
|--------|--------|
| تكلفة موحّدة | مادة واحدة = **نفس التكلفة** في كل المستودعات |
| أجور النقل | **إعادة تكليف** المادة — تكلفة جديدة تُطبَّق على **كل** المستودعات |
| الآلية | متوسط مرجح (أو نظام التكلفة §6) يُحدَّث عالمياً بعد مصاريف النقل |

```
فصل per مستودع:     مستودع أ (10) ──► مستودع ب (10) + مصاريف على ب فقط
بدون فصل + أجور:    إعادة تكليف المادة → تكلفة جديدة (12) في كل المستودعات
```

### 6.3) مراكز الكلف

| الخيار | المعنى |
|--------|--------|
| فصل التكلفة على أساس CC | تكلفة/مخزون يُتابَع per مركز كلف — يتقاطع مع [الحسابات](../الحسابات/README.md) |
| بدون فصل CC | CC للتحليل الإداري فقط — التكلفة على مستوى مادة/مستودع |

## 8) الربط بأنماط الفواتير

| من الإعداد العام | إلى الفاتورة |
|------------------|--------------|
| `inventory_method` | قالب **قيد اليومية** المُولَّد من النمط |
| `cost_per_warehouse` | تكلفة السطر من مستودع المصدر/الهدف |
| تكلفة إضافية على المناقلة | أسطر مصروف — **دوري ومستمر** §4 |

في [تعريف الفواتير](../الفواتير/تعريف-الفواتير.md):

- خيار النمط «اعتماد الجرد المستمر» → **يتبع الإعداد العام** — لا يُعارض القفل
- «تحرك المستودعات» → حركة كمية **دائماً**؛ شكل القيد يتبع §2

## 9) نموذج بيانات (مقترح)

```text
company_inventory_settings   -- id = 1
  inventory_method: periodic | perpetual
  costing_method: weighted_avg | fifo | standard | last_purchase
  cost_per_warehouse: boolean
  cost_per_cost_center: boolean
  track_quantity_on_movement: true          -- دائماً true
  foundation_locked: boolean
  foundation_locked_at, first_posted_inventory_at
```
