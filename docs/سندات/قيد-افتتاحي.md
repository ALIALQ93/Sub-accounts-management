# قيد افتتاحي

> **قرار معتمد:** **مطلوب** — ضمن نظام أنماط السندات غير المحدودة.  
> **آخر تحديث:** 2026-07-04 · **الحالة:** 📋 موثّق — لم يُنفَّذ

---

## 1) الهدف الواجب الوصول إليه

إدخال **الميزانية الافتتاحية** للمنشأة عند بدء التشغيل (أو بداية سنة مالية):

- أصول وخصوم وحقوق ملكية كاملة
- **منفصلة** عن حركات التشغيل في التقارير
- عمود **رصيد افتتاحي** في ميزان المراجعة

**النتيجة المرجوة:** بداية محاسبية صحيحة دون خلط الافتتاحي مع قبض/دفع/إغلاق حركات.

## 5) المواصفة المستهدفة

### 5.1) نمط السند

```text
voucher_pattern
  treat_as_journal = true
  is_opening_entry = true
  auto_post = true (موصى به)
  numbering_format = حسب السياسة (مثلاً OPEN-#YYYY)
```

**قيود النمط:**
- يُنشأ **مرة** أو مرات قليلة (سياسة: سند افتتاحي واحد per فرع/سنة)
- لا يرتبط بعميل/مورد إلزامياً
- لا تخصيصات إغلاق حركات على قيود افتتاحية

### 5.2) شاشة الإدخال

- جدول أسطر: حساب | مدين | دائن | CC (اختياري) | فرع (لاحقاً) | وصف
- **التوازن إلزامي:** مجموع مدين = مجموع دائن
- عملة السند: عملة أساس أو متعدد (قرار: افتراضياً عملة أساس)
- منع الحفظ إن وُجد سند افتتاحي **مرحّل** لنفس الفترة (إن طُبقت فترات)

### 5.3) الترحيل

```
سند افتتاحي (posted)
  → journal_entry مع is_opening_entry = true
  → journal_entry_lines عادية
```

### 5.4) التقارير

**ميزان المراجعة** (مستهدف):

| الحساب | رصيد افتتاحي | حركة الفترة (مدين) | حركة الفترة (دائن) | رصيد ختامي |
|--------|--------------|---------------------|---------------------|------------|

```
رصيد ختامي = افتتاحي + مدين − دائن
```

- حركة الفترة **تستثني** قيود `is_opening_entry`
- الرصيد الافتتاحي **يجمع** قيود الافتتاحي فقط

**كشف حساب:** خيار «يشمل الافتتاحي» أو رصيد افتتاحي كسطر أول.

## 6) نموذج بيانات مقترح

```text
journal_entries
  + is_opening_entry boolean default false

vouchers
  + is_opening_entry boolean default false
  -- أو عبر voucher_pattern.is_opening_entry

accounting_periods (لاحقاً)
  id, year, start_date, end_date, is_closed
```

**فهرس:** قيد افتتاحي واحد per `(company, branch?, period?)` — unique partial index.

## 7) قواعد التحقق

| # | القاعدة |
|---|---------|
| 1 | مدين = دائن |
| 2 | لا `voucher_allocations` على سند افتتاحي |
| 3 | بعد الترحيل: تعديل مقيد (admin) أو عكس فقط |
| 4 | لا حذف إن وُجدت حركة لاحقة تعتمد على الأرصدة (سياسة حذف) |
| 5 | حسابات الافتتاحي من دليل الحسابات النشط فقط |
