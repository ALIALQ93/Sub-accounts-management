# تعريف أنماط الفواتير

> **الموضوع:** تعريف أنماط الفواتير — مواصفة مستهدفة (مستندة على شرح المستخدم والصور).  
> **قرار:** أنماط **غير محدودة** — مثل [أنماط السندات](../سندات/تعريف-أنماط-السندات.md).  
> **الحالة:** ✅ **معتمد** — 2026-07-05 · تعديل **#27** 2026-07-05 (مناقلة ثنائية، فروع، أبعاد CC)  
> **نطاق الاعتماد:** النواة الكاملة — تبويبا **حجز** و**التخفيض** (تعريف النمط) **مؤجّلان** ولا يمنعان البناء.

---

## 1) الهدف من الأنماط — الفكرة الأساسية

### 1.1) ماذا تفعل الأنماط؟

**الهدف:** ضبط **الإعدادات الافتراضية** للفواتير — وليس فرض قالباً جامداً.

| البند | القرار |
|--------|--------|
| أنماط جاهزة | تُوفَّر عند التثبيت **للتسريع** — **ليست إلزامية** |
| تعديل الإعدادات | **مسموح دائماً** على مستوى النمط وعلى مستوى الفاتورة |
| إنشاء أنماط جديدة | غير محدود — حسب حاجة المنشأة |

**النتيجة:** النمط = **نقطة بداية** (حسابات، سلوك مواد، قيود…) يُسرِّع الإدخال؛ المستخدم يعدّل ما يلزم.

### 1.2) المحور الجوهري: إدخال أم إخراج

**الفرق الأساسي بين أي فاتورتين:**

```
                    ┌─────────────────┐
                    │     فاتورة      │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                             ▼
        ┌──────────┐                 ┌──────────┐
        │  إدخال   │                 │  إخراج   │
        └──────────┘                 └──────────┘
     زيادة مخزون/ذمم              نقص مخزون/إيراد
     مشتريات، مرتجع مبيعات       مبيعات، مرتجع مشتريات
     بضاعة أول المدة             صرف، تسوية إخراج
              │
              ▼
        ┌──────────┐
        │  مناقلة  │  عمليتان: إخراج عند الشحن → إدخال عند الاستلام
        └──────────┘  (مستند inventory_transfers يربط الفاتورتين)
```

| الاتجاه | معنى محاسبي/مخزني مختصر |
|---------|-------------------------|
| **إدخال** | ما **يدخل** للمخزون أو يُسجَّل كالتزام/تكلفة داخلة |
| **إخراج** | ما **يخرج** من المخزون أو يُسجَّل كإيراد/صرف |
| **مناقلة** | **ليس** فاتورة واحدة — **إخراج** (مصدر) ثم **إدخال** (هدف) عند الاستلام |

شراء/بيع/مناقلة/مرتجع/بضاعة أول المدة — تعابير تجارية تُضبط عبر إعدادات النمط (إدخال · إخراج).

### 1.3) الفكرة الفريدة: المواد + الحسابات

إعدادات النمط تجيب: **كيف تتعامل هذه الفاتورة مع المواد والحسابات؟**

| المحور | ما يحدده النمط |
|--------|----------------|
| **الحسابات** | مدين، دائن، مخزون، تكلفة، حسم — افتراضي + قابل للتعديل في الفاتورة |
| **المواد** | هل تتحرك المستودعات؟ أي سعر؟ أي تأثير على التكلفة؟ |
| **القيد** | هل يُولَّد سند قيد؟ آلياً؟ أي حسابات تُلمَس؟ |
| **مرجع** | هل تُحمَّل من فاتورة سابقة (مرتجع، إلغاء)؟ |
| **مركز الكلف** | على **رأس الفاتورة** و**كل سطر** — أساس المقاصة بين CC |
| **الفرع** | على **رأس الفاتورة** و**كل سطر** — أساس المقاصة بين الفروع ([ملف](../ملف/README.md)) |

هذا هو تمييز النمط عن «نوع» ثابت في الكود: **نفس الاتجاه (إدخال)** قد يتصرف محاسبياً ومخزنياً بشكل مختلف حسب الإعدادات.

### 1.4) مثال: المرتجع — مبيعات vs مشتريات

| النمط | الاتجاه | التأثير على المواد | التأثير على الحسابات |
|--------|---------|---------------------|------------------------|
| **مرتجع مبيعات** | **إدخال** | إعادة بضاعة للمخزون | عكس إيراد / ذمم عميل |
| **مرتجع مشتريات** | **إخراج** | إخراج من المخزون | عكس مشتريات / ذمم مورد |

**مرتجع المشتريات والتكلفة:** يؤثر على **كلفة المواد** حسب **نظام التكلفة** المعتمد في [إعدادات المواد العامة](../المواد%20والمستودعات/إعدادات-التكلفة.md) — يختاره المستخدم (متوسط مرجح، FIFO، معياري…).

### 1.5) بضاعة أول المدة

| البند | القرار |
|--------|--------|
| الإتاحة | **إلزامي** توفير نمط/إعداد **بضاعة أول المدة** |
| لمن | ليس للشركات الجديدة فقط — أي منشأة تبدأ تتبع مخزون أو تُدخل رصيد افتتاحي للمواد |
| الاتجاه | **إدخال** — دون شراء فعلي |
| الربط | [قيد افتتاحي](../سندات/قيد-افتتاحي.md) للحسابات؛ بضاعة أول المدة للمخزون |

### 1.6) الأنواع التشغيلية الأساسية: مبيعات، مشتريات، مناقلة

> **من شرح المستخدم:** ثلاثة أنماط تشغيلية جوهرية — تُوفَّر **جاهزة** عند التثبيت (قابلة للتعديل).

| النوع | `commercial_kind` | الاتجاه | الطرف | المخزون | الحسابات (مختصر) |
|--------|-------------------|---------|-------|---------|------------------|
| **مبيعات** | `sale` | **إخراج** | عميل | نقص من مستودع | إيراد / ذمم عميل أو نقد |
| **مشتريات** | `purchase` | **إدخال** | مورد | زيادة في مستودع | مشتريات / ذمم مورد أو نقد |
| **مناقلة — إخراج** | `transfer_out` | **إخراج** | — (داخلي) | نقص من مستودع المصدر | حسب الجرد + حساب وسيط (§1.6) |
| **مناقلة — إدخال** | `transfer_in` | **إدخال** | — (داخلي) | زيادة في مستودع الهدف | إتمام الاستلام + تكلفة إضافية |

#### مبيعات

| البند | القرار |
|--------|--------|
| الغرض | بيع بضاعة أو خدمة — **إخراج** من المخزون (إن وُجدت مواد) |
| الطرف | **عميل** — إلزامي غالباً (تبويب شروط §8) |
| آجل/نقدي | نمطان جاهزان أو نمط واحد بافتراض **آجل** + تعديل لنقدي |
| شروط الدفع | **ذات صلة** عند الآجل (§11) |
| إغلاق الحركات | فواتير آجل → حركات مفتوحة على ذمم العميل |

#### مشتريات

| البند | القرار |
|--------|--------|
| الغرض | شراء بضاعة أو خدمة — **إدخال** للمخزون |
| الطرف | **مورد** |
| التكلفة | يُحدَّث رصيد التكلفة حسب [إعدادات التكلفة](../المواد%20والمستودعات/إعدادات-التكلفة.md) |
| شروط الدفع | للشراء الآجل — تسديد للمورد خلال N يوم |
| إغلاق الحركات | فواتير شراء آجل → ذمم دائنة مفتوحة |

#### مناقلة — إخراج ثم إدخال (قرار #27)

> **لا نقل مباشر** في فاتورة واحدة. الأفضل: **فاتورة إخراج** عند الشحن، ثم **فاتورة إدخال** عند الاستلام — مع إمكان **استلام جزئي**.

| البند | القرار |
|--------|--------|
| مستند الربط | `inventory_transfers` يربط فاتورة OUT بفاتورة IN |
| عند الشحن | فاتورة **إخراج** (`transfer_out`) — فرع/مستودع المصدر |
| عند الاستلام | فاتورة **إدخال** (`transfer_in`) — فرع/مستودع الهدف |
| استلام جزئي | `qty_received < qty_shipped` — الباقي يبقى `in_transit` |
| الطرف | **بدون** عميل/مورد خارجي |
| التكلفة الإضافية | تبويب **حسابات إضافية** على OUT و/أو IN (نقل، أجور…) — §13.3 |
| حساب وسيط «بالطريق» | **مرن per نمط** — يحدده المستخدم في إعدادات نمط المناقلة (§6.2)؛ ليس إلزامياً على مستوى الشركة |
| القيد | يتبع `inventory_method` — شكل مختلف بين الدوري والمستمر (§1.7) |
| آجل / شروط دفع | **لا ينطبق** |
| إغلاق الحركات | **لا ذمم طرفية** — لكن أسطر القيد تحمل `branch_id` + `cost_center_id` للمقاصة لاحقاً |

```
inventory_transfer (draft)
    │
    ├─► فاتورة إخراج (transfer_out) — ترحيل عند الشحن
    │       مخزون مصدر ↓  →  حساب وسيط (إن وُجد في النمط)
    │
    └─► فاتورة إدخال (transfer_in) — ترحيل عند الاستلام
            حساب وسيط ↓  →  مخزون هدف ↑
            + حسابات إضافية (مصاريف نقل/أجور)
```

| المرحلة | جرد **مستمر** (مختصر) | جرد **دوري** (مختصر) |
|---------|----------------------|----------------------|
| إخراج | مدين وسيط/بالطريق — دائن مخزون مصدر | تتبع كمية + قيد حسب قالب النمط |
| إدخال | مدين مخزون هدف — دائن وسيط/بالطريق | إدخال + مصاريف منفصلة §4.2 |

### 1.7) توليد القيود والجرد (دوري / مستمر)

> **مصدر الحقيقة:** [إعدادات المخزون العامة](../المواد%20والمستودعات/إعدادات-المخزون-العامة.md) — **قفل عند أول إعداد**.

| البند | القرار |
|--------|--------|
| توليد القيود | ينعكس وفق `inventory_method`: **دوري** أو **مستمر** |
| تتبع الكميات | **مع كل حركة** حتى في الجرد الدوري — احتساب مخزون نظامي دائماً |
| الفرق الجوهري | **شكل قيد اليومية** — خاصة **مبيعات** و**مناقلة** |

| العملية | جرد **مستمر** | جرد **دوري** |
|---------|---------------|--------------|
| **مبيعات** | إيراد/ذمم + **تكلفة مبيعات ↔ مخزون** فوراً | إيراد/ذمم **فقط** — تكلفة آخر الفترة |
| **مناقلة (out/in)** | قيدان منفصلان حسب المرحلة | كمية + مصاريف (كلا النظامين) |
| **مشتريات** | مدين مخزون | مدين مشتريات (مصروف) — حسب السياسة |

**فصل التكلفة (قفل):** per مستودع و/أو per مركز كلف — يحدد هل تختلف تكلفة المادة بين المستودعات، وهل أجور النقل **تعيد تكليف** المادة في كل المستودعات. التفاصيل: [§5 إعدادات المخزون](../المواد%20والمستودعات/إعدادات-المخزون-العامة.md).

> **مرتجعات** و**بضاعة أول المدة** أنماط إضافية — §1.4، §1.5، §12.2.

## 2) الهدف التقني الواجب الوصول إليه

تمكين **عدد غير محدود من أنماط الفواتير**، كل نمط يضبط افتراضيات قابلة للتعديل:

- محور **إدخال / إخراج**
- تفاعل **المواد** (مخزون، تكلفة، تسعير) و**الحسابات** (قيود، ذمم)
- مرجع، CC، قيد — أساس [إغلاق الحركات](../سندات/إغلاق-الحركات.md)

## 6) التبويب 1: خصائص النمط

### 6.1) حقول التعريف الأساسية

| الحقل | الوصف | التأثير | حالة عندنا | معيار الاكتمال |
|--------|--------|---------|--------------|----------------|
| رقم النمط | تسلسلي تلقائي | معرّف | ⚪ | `invoice_patterns.pattern_no` |
| اسم النمط | عربي | قوائم، تقارير | ⚪ | `name_ar` |
| الاسم اللاتيني | i18n | — | ⚪ | `name_en` |
| درجة سرية الفاتورة | منخفضة → عالية | صلاحيات | ⚪ | `secrecy_level` |
| العملة | افتراضية | أسعار الفاتورة | ⚪ | `default_currency_id` |
| **الفرع** | افتراضي للنمط | رأس الفاتورة والأسطر | ⚪ | `default_branch_id` |
| مركز الكلفة | افتراضي للنمط | يُورَّث للأسطر — قابل للتعديل per سطر | ⚪ | `default_cost_center_id` |

### 6.2) حسابات افتراضية (قابلة للتعديل في الفاتورة)

> **قرار معتمد:** مثل السندات — افتراض في النمط + **تعديل في مستند الفاتورة** بصلاحية.

| الحقل | الوصف | مثال المشتريات |
|--------|--------|----------------|
| حساب الدائن | الطرف الدائن في القيد | صندوق |
| حساب المدين | الطرف المدين | مشتريات خارجية |
| حساب التكلفة | تكلفة البضاعة | — |
| حساب المخزون | عند حركة مخزون | — |
| حساب الحسميات | خصومات مكتسبة | الحسم المكتسب |
| حساب الإضافي | إضافات على الفاتورة | — |
| حساب عمولة المندوب | عمولات | — |
| **حساب وسيط المناقلة** | بضاعة بالطريق / تسوية داخلية — **اختياري per نمط** | يحدده المستخدم؛ مرن بين الدوري والمستمر |
| **حساب تسوية بين فروع** | افتراضي لمناقلة بين فروع — لاحقاً للمقاصة | يُورَّث ويُعدَّل في الفاتورة |
| **حساب تسوية بين CC** | افتراضي — لاحقاً لإغلاق الحركات | يُورَّث ويُعدَّل في الفاتورة |
| سند القيد | نمط القيد المُولَّد | سند قيد |
| نمط إدخال المناقلة | ربط `transfer_out` بـ `transfer_in` | `paired_input_pattern_id` |

### 6.3) فروع ومستودعات

| الحقل | الوصف |
|--------|--------|
| **الفرع الافتراضي** | `default_branch_id` — يُحمَّل على رأس الفاتورة والأسطر |
| المستودع الافتراضي | مستودع **المصدر** (مبيعات/إخراج مناقلة) أو **الوجهة** (مشتريات/إدخال مناقلة) |
| المستودع الهدف | لأنماط **مناقلة إخراج** — مستودع الشحن؛ ولـ **إدخال** — مستودع الاستلام |

> **CC مستقل عن الفرع** (قرار #27): مركز الكلف لا يُربَط إلزامياً بفرع — كلاهما أبعاد منفصلة على الرأس والأسطر.  
> المستودع يتبع فرعاً (`warehouses.branch_id`) — يُتحقَّق من اتساق `line.branch_id` مع المستودع.

### 6.4) المحور الجوهري: إدخال / إخراج

| الحقل | القيم | ملاحظة |
|--------|-------|--------|
| **`direction`** | **`input`** · **`output`** | **إلزامي** — إدخال أو إخراج (المناقلة = نمطان منفصلان) |
| `commercial_kind` | `sale` · `purchase` · `transfer_out` · `transfer_in` · `return_*` · `opening_stock` · … | تصنيف تجاري |

**أمثلة:**

| التصنيف التجاري | الاتجاه | سلوك مختصر |
|-----------------|---------|------------|
| مشتريات | إدخال | مخزون ↑، ذمم مورد |
| مبيعات | إخراج | مخزون ↓، إيراد/ذمم عميل |
| **مناقلة إخراج** | **إخراج** | نقص مصدر — وسيط اختياري |
| **مناقلة إدخال** | **إدخال** | زيادة هدف — إتمام الاستلام |
| مرتجع مبيعات | **إدخال** | إعادة للمخزون |
| مرتجع مشتريات | **إخراج** | خروج من المخزون + **تأثير تكلفة** |
| بضاعة أول المدة | **إدخال** | رصيد افتتاحي مواد |

### 6.5) خيارات السلوك (checkboxes)

| الخيار | الوصف | مثال المشتريات | أولوية |
|--------|--------|----------------|--------|
| تولد سند قيد | إنشاء قيد عند الترحيل | ✅ مفعّل | **إلزامي للنواة** |
| تولد سند قيد بشكل آلي | ترحيل فوري | ✅ مفعّل | إلزامي |
| مركز الكلفة يخص حساب البضاعة | CC على طرف البضاعة | ✅ | **إلزامي لسيناريو ز** |
| مركز الكلفة يخص حساب المورد/العميل | CC على طرف الطرف | ✅ | إلزامي |
| تحميل عملة العميل | عملة من بطاقة الطرف | ✅ | موصى به |
| تطبيق حسومات نقاط المبيع | ربط POS | ✅ | لاحقاً |
| تحرك المستودعات | حركة مخزنية (كمية دائماً) | ☐ | **إلزامي للمناقلة** — القيد يتبع [إعدادات الجرد](../المواد%20والمستودعات/إعدادات-المخزون-العامة.md) |
| جرد مستمر في القيد | يتبع **الإعداد العام** — ليس override per نمط | — | `inventory_method` من الشركة |
| تفعيل شروط الدفع | استحقاق من النمط | ☐ | §11 — [عميل](../عميل/المزايا.md) |
| تعامل كفاتورة كمرتجع | سلوك مرتجع + مرجع | ☐ | نمط منفصل لكل اتجاه |
| تعامل كفاتورة أول المدة | بضاعة افتتاحية | ☐ | **إلزامي توفير النمط** — §1.5 |

### 6.6) أنماط التسعير (أسفل الشاشة)

| الحقل | مثال المشتريات |
|--------|----------------|
| نمط تسعير المواد | سعر الشراء |
| سعر التكلفة | سعر الشراء |
| نمط تسعير المواد المستهلكة | سعر المبيع |

### 6.7) الحالة الافتراضية: آجل / نقدي

| الحقل | القيم | الوصف |
|--------|-------|--------|
| **`default_settlement_mode`** | **`credit`** · **`cash`** | الحالة الافتراضية عند إنشاء فاتورة بهذا النمط |

| الوضع | المعنى | التأثير |
|--------|--------|---------|
| **آجل** (`credit`) | ذمم — تسديد لاحق | حركة مفتوحة → [إغلاق الحركات](../سندات/إغلاق-الحركات.md)؛ **شروط الدفع** ذات صلة |
| **نقدي** (`cash`) | تسديد فوري | حساب نقد/بنك في القيد؛ عادة **بدون** ذمم مفتوحة |

- يُحدَّد في **النمط** كافتراض — ويُعدَّل في الفاتورة عند الحاجة.
- أنماط **البيع/الشراء الآجل** غالباً: افتراض = **آجل**.
- أنماط **البيع/الشراء النقدي** أو نقاط البيع: افتراض = **نقدي**.

## 7) التبويب 2: مرجع

يتحكم بربط الفاتورة ب**فاتورة/مستند مرجعي** — أساسي للمرتجعات والإلغاء وإغلاق الحركات.

### 7.1) خيارات مفعّلة في المثال (مستهدفة)

| الخيار | الوصف | أهمية |
|--------|--------|--------|
| قفل الفاتورة عند تحميلها كمرجع | منع تعديل المرجع بعد الربط | تكامل |
| مطابقة الفاتورة مع المرجع | تحقق تناسق البيانات | جودة |
| السماح بالتحميل الجزئي | جزء من أسطر المرجع فقط | **مرتجعات جزئية** |
| عدم عرض المراجع المنتهية | إخفاء المغلقة/المنتهية | UX |
| تحميل المستودع من المرجع | — | مخزون |
| تحميل العميل/المورد من المرجع | — | إلزامي غالباً |
| تحميل المواد من المرجع | أسطر المرجع | مرتجعات |
| تحميل مركز الكلفة من المرجع | CC per سطر | **سيناريو ز** |
| تحميل مواد الخدمات | أصناف خدمية | — |
| تحميل السعر من المرجع | سعر المرجع | تسعير |
| تحميل تاريخ الصلاحية | صلاحية | مواد |
| تحميل طريقة الدفع | شروط الدفع | استحقاق |

### 7.2) خيارات غير مفعّلة (متاحة للتعريف)

| الخيار | الاستخدام المحتمل |
|--------|-------------------|
| حذف المرجع | إزالة الربط |
| ربط آلي | ربط تلقائي عند الإنشاء |
| تحديث الأسعار من المرجع | مزامنة أسعار |
| عرض فواتير الفرع فقط | [فروع](../ملف/README.md) |
| إجبار استخدام المرجع | لا فاتورة بدون مرجع |
| مراجع متعددة | أكثر من فاتورة مصدر |
| إغلاق المرجع يدوياً | إغلاق صريح للمرجع |
| استخدام نفس رقم المرجع | ترقيم موحّد |
| تحميل السعر الصافي | بعد خصم |
| تحميل رقم الإيصال / التاريخ | حقول إضافية |
| تحميل الحسم / الإضافي | خصومات المرجع |

### 7.3) حد زمني للمرجع

```
تحميل المرجع خلال [N] يوماً
```

مثال المستند: `0` = بدون حد (أو يُفسَّر كـ«الكل» — يُحدَّد عند التنفيذ).

## 8) التبويب 5: شروط

يتحكم بـ**إلزاميات الحقول** قبل حفظ الفاتورة — مثل [شروط السندات](../سندات/تعريف-أنماط-السندات.md).  
عند المخالفة: **رسالة تنبيه** — لا حفظ حتى استيفاء الشرط.

### 8.1) الإجبار على الاختيار (checkboxes)

| الشرط | الوصف | مثال المستند | أولوية |
|--------|--------|--------------|--------|
| الإجبار على اختيار **العميل** | الطرف (عميل/مورد) إلزامي | ☐ | إلزامي لأنماط البيع/الشراء الآجل |
| الإجبار على اختيار **المندوب** | مندوب مبيعات | ☐ | لاحقاً — [مندوب](../مندوب/README.md) |
| الإجبار على اختيار **مركز الكلفة** | CC إلزامي على الفاتورة/الأسطر | **✅** | **إلزامي لسيناريو ز** |
| الإجبار على اختيار **رقم الإيصال** | حقل إيصال خارجي | ☐ | اختياري per نمط |
| **منع تكرار رقم الإيصال** | تحقق فريدية — ليس «إجبار اختيار» | ☐ | تحقق DB + UI |
| الإجبار على اختيار **شروط الدفع** | استحقاق/طريقة دفع | ☐ | يرتبط [عميل](../عميل/المزايا.md) وإغلاق الحركات |
| الإجبار على اختيار **المستودع** | مستودع إلزامي | ☐ | عند تفعيل المخزون |
| الإجبار على اختيار **اللون** | صفة مادة (متغير) | ☐ | مرحلة مواد — أصناف بصفات |
| الإجبار على اختيار **الحجم** | صفة مادة (متغير) | ☐ | مرحلة مواد |
| الإجبار على اختيار **المصدر** | مصدر التوريد/البيع | ☐ | حسب قطاع المنشأة |
| الإجبار على اختيار **العيار** | عيار/نقاء (مجوهرات…) | ☐ | قطاعي — اختياري |

> **ملاحظة:** اللون/الحجم/المصدر/العيار = صفات **أسطر المواد**؛ الإجبار يُطبَّق per سطر عند وجود مادة بصفات.

### 8.2) أزرار التخصيص

| الزر | الوظيفة المتوقعة |
|------|-------------------|
| **تخصيص المواد** | تحديد **قائمة مواد مسموحة** في هذا النمط (أو استثناءات) |
| **تخصيص الأصناف** | تحديد **مجموعات/أصناف** مسموحة — يُورَّث للمواد التابعة |

```
نمط فاتورة
    → (اختياري) تخصيص أصناف مسموحة
    → (اختياري) تخصيص مواد مسموحة
    → عند الإدخال: رفض مادة خارج القائمة أو خارج الصنف المسموح
```

### 8.3) سلوك التحقق

| المرحلة | السلوك |
|---------|--------|
| حفظ مسودة | يُطبَّق الإجبار حسب إعداد النمط (أو يُخفَّف لمسودة — يُحدَّد عند التنفيذ) |
| اعتماد / ترحيل | **كل الشروط المفعّلة** إلزامية |
| رسالة الخطأ | اسم الحقل الناقص بالعربية — مثل السندات |

### 8.4) نموذج بيانات (شروط)

```text
invoice_pattern_conditions
  pattern_id
  require_party boolean              -- عميل/مورد
  require_sales_rep boolean
  require_cost_center boolean
  require_receipt_no boolean
  prevent_duplicate_receipt_no boolean
  require_payment_terms boolean
  require_warehouse boolean
  require_color boolean
  require_size boolean
  require_source boolean
  require_caliber boolean

invoice_pattern_allowed_materials    -- من «تخصيص المواد»
  pattern_id, material_id

invoice_pattern_allowed_categories   -- من «تخصيص الأصناف»
  pattern_id, category_id
```

## 9) التبويب 3: تدوير (تقريب القيم)

> **ليس** إقفال فترة محاسبية — **تدوير** = **تقريب** مبالغ الفاتورة لتناسب **فئات العملة** المطروحة في السوق.

### 9.1) الهدف

| البند | القرار |
|--------|--------|
| المشكلة | أسعار/مجاميع قد تنتج كسوراً غير قابلة للتداول (مثلاً 1,234.67) |
| الحل | تقريب الإجمالي (أو الأسطر) لأقرب **وحدة عملة سائدة** |
| المصدر | فئات العملة من [العملات](../الحسابات/README.md) — أصغر فئة تداول per عملة |

**أمثلة:**

| عملة | فئة شائعة | قبل | بعد التدوير |
|------|-----------|-----|-------------|
| دينار | 250 دينار | 1,234,567 | 1,234,500 أو 1,234,750 |
| دولار | 0.05 | 99.97 | 100.00 |

### 9.2) إعدادات النمط

| الحقل | الوصف |
|--------|--------|
| `rounding_enabled` | تفعيل التدوير على هذا النمط |
| `rounding_target` | `invoice_total` · `line_amount` · `both` |
| `rounding_mode` | `nearest` · `up` · `down` |
| `rounding_step` | خطوة التقريب — افتراض من العملة أو override |

> الفرق بين التدوير والخصم: التدوير **تقني/عملة**؛ الخصم **تجاري** (تبويب التخفيض).

### 9.3) نموذج بيانات

```text
rounding_enabled boolean
rounding_target enum
rounding_mode enum
rounding_step numeric nullable    -- null = من جدول العملة
```

## 10) التبويب 4: ترقيم

> **ليس مشكلة تقنية** — نفس فكرة [ترقيم السندات](../سندات/تعريف-أنماط-السندات.md): كل نمط **تسلسل مستقل**.

### 10.1) أوضاع الترقيم

| الوضع | الوصف | مثال |
|--------|--------|------|
| **تسلسل بسيط** | رقم متزايد فقط | `1`، `2`، `3`… |
| **تسلسل + لاحقة** | بادئة/تاريخ/رمز ثابت | `INV-2026-0001`، `(# YYYYMMDD)` |

| الحقل | الوصف |
|--------|--------|
| `numbering_start` | رقم بداية التسلسل |
| `numbering_format` | قالب العرض — لاحقة تاريخ أو بادئة نمط |
| `numbering_reset` | `never` · `yearly` · `monthly` — اختياري |

**الكود المرجعي الحالي:** `voucher_number_sequences`، `format-voucher-no.ts` — يُعاد استخدامه للفواتير.

## 11) شروط الدفع

يحدد **متى يُتوقَّع التسديد** — في **الأغلب** لفواتير **البيع والشراء الآجل**.

### 11.1) القاعدة العامة

| البند | القرار |
|--------|--------|
| الاستخدام الشائع | **تسديد خلال فترة محددة** — مثل 30، 60، **90** يوماً |
| النطاق | بيع آجل + شراء آجل |
| النقدي | عادة **بدون** استحقاق — أو 0 يوم |
| الافتراض | من **النمط**؛ يُحمَّل من **بطاقة العميل/المورد** إن وُجد |

### 11.2) مصادر الاستحقاق (أولوية عند التنفيذ)

```
1. شروط الدفع على الفاتورة (إن عُدِّلت يدوياً)
2. افتراض النمط (مثلاً 90 يوماً)
3. أيام الاستحقاق على العميل/المورد — [عميل](../عميل/المزايا.md)
```

| الحقل | مثال | الوصف |
|--------|------|--------|
| `default_payment_terms_days` | 90 | عدد أيام التسديد من تاريخ الفاتورة |
| `payment_terms_enabled` | true | تفعيل حساب `due_date` |
| `due_date` | محسوب | `invoice_date + payment_terms_days` |

### 11.3) الربط

| النظام | الاستخدام |
|--------|-----------|
| [إغلاق الحركات](../سندات/إغلاق-الحركات.md) | «مؤهل للتسديد» / «لم يحن» / «متأخر» |
| تبويب **شروط** (§8) | «الإجبار على اختيار شروط الدفع» — إلزام قبل الحفظ |
| خصائص (§6.7) | آجل = شروط الدفع **ذات صلة**؛ نقدي = غالباً لا |

### 11.4) أمثلة أنماط

| النمط | آجل/نقدي | أيام افتراضية |
|--------|----------|----------------|
| بيع آجل | آجل | 90 |
| شراء آجل | آجل | 60 |
| بيع نقدي | نقدي | — |
| مرتجع | حسب المرجع | من الفاتورة المرجعية |
| مناقلة out/in | — | لا ينطبق |

## 12) الأنماط الجاهزة عند التثبيت

> قابلة للتعديل — **ليست إلزامية** الاستخدام؛ تُسرِّع البدء.

### 12.1) الثلاثة الأساسية (معتمدة)

| # | اسم النمط المقترح | `commercial_kind` | `direction` | افتراضات رئيسية |
|---|-------------------|-------------------|-------------|-----------------|
| 1 | **مبيعات** | `sale` | `output` | عميل، إخراج مخزون، آجل أو نقدي per نسخة |
| 2 | **مشتريات** | `purchase` | `input` | مورد، إدخال مخزون، شروط دفع للآجل |
| 3 | **مناقلة — إخراج** | `transfer_out` | `output` | فرع/مستودع مصدر، حساب وسيط اختياري |
| 4 | **مناقلة — إدخال** | `transfer_in` | `input` | فرع/مستودع هدف، يُكمّل `inventory_transfers` |

**نسخ شائعة (اختيارية عند التثبيت):**

| من الأساسي | نسخة | الفرق |
|------------|------|-------|
| مبيعات | مبيعات آجل | `credit` + 90 يوم |
| مبيعات | مبيعات نقدي | `cash` |
| مشتريات | مشتريات آجل | `credit` + شروط دفع |
| مشتريات | مشتريات نقدي | `cash` |

### 12.2) أنماط إضافية مطلوبة (ليست الثلاثة — لكن إلزامي التوفر)

| النمط | السبب |
|--------|--------|
| مرتجع مبيعات | §1.4 — إدخال + مرجع |
| مرتجع مشتريات | §1.4 — إخراج + تكلفة |
| بضاعة أول المدة | §1.5 — إلزامي |

### 12.3) بذور التثبيت (seed)

```text
invoice_pattern_seeds:
  - مبيعات (sale, output)
  - مشتريات (purchase, input)
  - مناقلة إخراج (transfer_out, output)
  - مناقلة إدخال (transfer_in, input)
  - مرتجع مبيعات
  - مرتجع مشتريات
  - بضاعة أول المدة
  -- + نسخ آجل/نقدي حسب قالب المنشأة
```

## 13) مستند الفاتورة — تبويبان للإدخال

> **من تصميم المستخدم:** شاشة **إدخال الفاتورة** (التشغيل) — ليست تبويبات **تعريف النمط** (§5).

### 13.1) الهيكل

```
┌─────────────────────────────────────────────────┐
│  رأس الفاتورة: نمط، فرع، CC، طرف، تاريخ، آجل/نقدي…  │
├─────────────────────────────────────────────────┤
│  [ تبويب المواد ]  │  [ تبويب الحسابات الإضافية ] │
├─────────────────────────────────────────────────┤
│  إجماليات · حفظ · اعتماد · ترحيل                │
└─────────────────────────────────────────────────┘
```

| التبويب | الغرض |
|---------|--------|
| **المواد** | أصناف بكميات وأسعار — حركة مخزون وتكلفة |
| **الحسابات الإضافية** | أسطر محاسبية **بدون مادة** — مصاريف، خدمات، إضافات |

### 13.2) تبويب المواد

| الحقل | إلزامي | ملاحظة |
|--------|--------|--------|
| المادة | حسب النمط/الشروط | من «تخصيص المواد» §8 |
| الكمية | ✅ | يحرّك المخزون؛ للمناقلة: `qty_shipped` / `qty_received` |
| الوحدة | ✅ | من **وحدات المادة** — `material_unit_id` ([وحدات-المواد](../المواد%20والمستودعات/وحدات-المواد.md)) |
| السعر | ✅ | من نمط التسعير §6.6 |
| **الفرع** | ✅ | per سطر — يتسق مع المستودع |
| مستودع | عند المخزون | مصدر (out) أو هدف (in) |
| **مركز الكلف** | حسب الشروط §8 | **per سطر** — أساس سيناريو ز |
| صفات (لون، حجم…) | حسب الشروط | §8 |

### 13.2.1) رأس الفاتورة — أبعاد إلزامية

| الحقل | الوصف |
|--------|--------|
| `branch_id` | فرع إصدار المستند |
| `cost_center_id` | CC افتراضي — يُنسَخ للأسطر الجديدة |
| حسابات override | من النمط — قابلة للتعديل |
| `inventory_transfer_id` | عند المناقلة — ربط بمستند النقل |
| `transfer_role` | `out` · `in` — عند المناقلة فقط |

**ينطبق على:** مبيعات، مشتريات، مناقلة (كميات منقولة)، مرتجعات، بضاعة أول مدة.

### 13.3) تبويب الحسابات الإضافية

| الحقل | إلزامي | ملاحظة |
|--------|--------|--------|
| الحساب | ✅ | من دليل الحسابات |
| المبلغ | ✅ | مدين أو دائن حسب النمط والاتجاه |
| البيان | موصى | وصف السطر |
| **الفرع** | ✅ | per سطر |
| **مركز الكلف** | حسب الشروط | CC على المصروف/الإيراد |

**أمثلة الاستخدام:**

| الحالة | مثال |
|--------|------|
| **مناقلة** + تكلفة نقل | حساب مصروف نقل — يُحمَّل على التكلفة ([§4](../المواد%20والمستودعات/إعدادات-المخزون-العامة.md)) |
| **مناقلة** + أجور عمال | حساب أجور — إعادة تكليف حسب الإعداد العام |
| خدمة بلا مخزون | تركيب، صيانة — حساب إيراد/مصروف |
| إضافات/خصومات | حساب الإضافي أو الحسم من النمط §6.2 |
| عمولة مندوب | حساب عمولة المندوب |

> التبويب **اختياري** — فاتورة مواد فقط تُكتفى بالتبويب الأول.

### 13.4) الترحيل والقيد

```
تبويب المواد        ──┐
                       ├──► تجميع ──► سند قيد واحد (حسب النمط + الجرد العام)
تبويب حسابات إضافية ──┘
```

| البند | القرار |
|--------|--------|
| قيد واحد | **نعم** — سطران من مصدرين، مستند واحد |
| ترتيب الأسطر | مواد ثم حسابات (أو حسب قالب القيد) |
| الإجمالي | مجموع التبويبين + تدوير §9 |

### 13.5) نموذج بيانات (أسطر)

  branch_id, cost_center_id
  material_id, quantity, material_unit_id, unit_price

## 15) سلوك ما بعد التعريف

```
تعريف النمط (حفظ)
    → إنشاء فاتورة بهذا النمط
    → (اختياري) تحميل مرجع — جزئي/كامل
    → تبويب **المواد** (+ اختياري: **حسابات إضافية**)
    → أسطر مواد + أسطر حسابات → قيد مجمّع
    → حفظ / اعتماد
    → [إن آلي] توليد سند قيد
    → حالة آجل/نقدي (افتراض النمط — لا ينطبق على مناقلة)
    → (إن آجل: مبيعات/مشتريات) due_date + حركة مفتوحة
    → (مناقلة: مصدر→هدف — بلا ذمم)
    → لاحقاً: قبض/دفع عادي أو إغلاق حركات (مساران منفصلان)
```
