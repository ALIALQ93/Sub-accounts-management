# مخطط النظام — وثائق التطوير

هذا المجلد هو **المرجع الرسمي** لهدف النظام، ميزات كل قسم، علاقاتها، وحالة التقدم.
يُحدَّث مع كل تطوير — لا ننفّذ ميزة دون توثيق هدفها ومعيار اكتمالها أولاً.

## الهدف العام للنظام

بناء نظام إدارة حسابات فرعية (ERP محاسبي) يغطي:

1. **المرجعيات** — حسابات، عملات، أطراف، مواد، فروع
2. **المعاملات** — سندات، فواتير، مخزون
3. **النتائج المحاسبية** — قيود يومية، ذمم، تقارير، قوائم مالية
4. **الحوكمة** — صلاحيات، تدقيق، إقفال فترات

الهيكل مأخوذ من مواصفة أقسام معتمدة؛ نبني تدريجياً حسب الأولوية.

## كيف نستخدم هذه الوثائق

### عند التخطيط لميزة جديدة

1. افتح مجلد القسم → `المزايا.md`
2. أضف أو حدّث صف الميزة (الوصف، العلاقات، معيار الاكتمال)
3. حدّث `README.md` للقسم إن تغيّر الهدف أو العلاقات
4. حدّث `PROGRESS.md` بحالة القسم

### عند إنهاء تطوير

1. غيّر **الحالة** في `المزايا.md` إلى ✅
2. اذكر **أين في الكود** (مسار الملف أو جدول DB)
3. حدّث `PROGRESS.md` و`CHANGELOG.md`

### عند شرح ميزات من المستخدم

انسخ قالب `_template/ميزة.md` أو أضف صفاً في `المزايا.md` بالتفاصيل التي يقدّمها المستخدم.
